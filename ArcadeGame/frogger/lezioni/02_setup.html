Setup dell'Ambiente di Sviluppo

Introduzione

Prima di iniziare a sviluppare il nostro gioco Frogger, dobbiamo configurare un ambiente di sviluppo adeguato. In questa lezione, esploreremo gli strumenti necessari e come configurarli per un flusso di lavoro efficiente.

Strumenti Necessari

Per sviluppare il nostro gioco Frogger con JavaScript, avremo bisogno dei seguenti strumenti:

Editor di Testo

Un buon editor di testo è fondamentale per scrivere codice in modo efficiente. Consigliamo Visual Studio Code (VS Code), un editor gratuito, open-source e altamente personalizzabile con un eccellente supporto per JavaScript.

VS Code offre funzionalità come l'evidenziazione della sintassi, il completamento automatico, il debugging integrato e un'ampia gamma di estensioni che possono migliorare la tua esperienza di sviluppo.

Scarica VS Code

Browser Web

Avrai bisogno di un browser web moderno per testare il tuo gioco. Consigliamo Google Chrome o Mozilla Firefox, entrambi dotati di potenti strumenti di sviluppo che ti aiuteranno a debuggare il tuo codice JavaScript.

Gli strumenti di sviluppo del browser (DevTools) ti permettono di ispezionare il DOM, debuggare il JavaScript, monitorare le prestazioni e molto altro.

Scarica Chrome Scarica Firefox

Server Web Locale (Opzionale)

Sebbene non sia strettamente necessario per iniziare, un server web locale può essere utile per testare alcune funzionalità che richiedono un ambiente server. Ci sono diverse opzioni semplici:

  • Live Server: Un'estensione di VS Code che avvia un server di sviluppo locale con funzionalità di ricarica automatica.
  • Node.js con http-server: Se hai già Node.js installato, puoi utilizzare il pacchetto http-server.
  • Python SimpleHTTPServer: Se hai Python installato, puoi utilizzare il suo server HTTP integrato.

Configurazione dell'Ambiente

1. Installazione di Visual Studio Code

  1. Scarica VS Code dal sito ufficiale.
  2. Segui le istruzioni di installazione per il tuo sistema operativo.
  3. Una volta installato, avvia VS Code.

2. Estensioni Consigliate per VS Code

VS Code può essere migliorato con estensioni. Ecco alcune estensioni utili per lo sviluppo JavaScript:

  • Live Server: Per avviare un server di sviluppo locale con ricarica automatica.
  • ESLint: Per identificare e correggere problemi nel tuo codice JavaScript.
  • Prettier: Per formattare automaticamente il tuo codice.
  • JavaScript (ES6) code snippets: Per accelerare la scrittura del codice con snippet predefiniti.

Per installare un'estensione in VS Code:

  1. Clicca sull'icona delle estensioni nella barra laterale (o premi Ctrl+Shift+X).
  2. Cerca l'estensione desiderata.
  3. Clicca su "Install".

Configurazione di Live Server

Dopo aver installato l'estensione Live Server, puoi avviare un server locale seguendo questi passaggi:

  1. Apri la cartella del tuo progetto in VS Code.
  2. Clicca con il tasto destro sul file HTML che vuoi visualizzare (ad esempio, index.html).
  3. Seleziona "Open with Live Server" dal menu contestuale.
  4. Il tuo browser predefinito si aprirà automaticamente con la pagina visualizzata.

3. Configurazione degli Strumenti di Sviluppo del Browser

Gli strumenti di sviluppo del browser sono essenziali per il debug del tuo codice JavaScript. Ecco come accedervi:

  • In Chrome: Premi F12 o Ctrl+Shift+I (o Cmd+Option+I su Mac).
  • In Firefox: Premi F12 o Ctrl+Shift+I (o Cmd+Option+I su Mac).

Familiarizza con le diverse schede degli strumenti di sviluppo:

  • Elements/Inspector: Per esaminare e modificare il DOM.
  • Console: Per eseguire comandi JavaScript e visualizzare i log.
  • Sources/Debugger: Per impostare breakpoint e debuggare il codice.
  • Network: Per monitorare le richieste di rete.

Creazione della Struttura del Progetto

Ora che abbiamo configurato il nostro ambiente di sviluppo, possiamo creare la struttura di base del nostro progetto Frogger. Nella prossima lezione, esploreremo in dettaglio come organizzare i file e le cartelle del nostro progetto.

Consiglio: Prima di iniziare a scrivere codice, è una buona pratica pianificare la struttura del tuo progetto. Questo ti aiuterà a mantenere il codice organizzato e facile da gestire man mano che il progetto cresce.
